Structures and configurations

This section contains functions working with structures or magnetic configurations.

Sym4state.Utils.mag_config — Function

This function yields an array with size of [3, mag_count, 36] that comprises all 36 magnetic configurations utilized in the computation of the interaction matrix for a given number of magnetic atoms mag_count and the initial norm value s_value of MAGMOM.

The resulting configurations pertain to the interaction between two atoms specified by the target_idx_vec vector.

mag_config(
    mag_count::Int64,
    target_idx_vec::Vector{Int64},
    s_value::Float64
) -> Array{Float64, 3}
source
Sym4state.Utils.py_struc_to_struc — Function

This function transforms the Structure object from Pymatgen into a Struc object.

The distinct identifier uni_num will be supplied to the Struc object. The spin_mat variable represents the spin value assigned to each atom in the structure.

py_struc_to_struc(
    py_struc;
    uni_num,
    spin_mat
) -> Sym4state.Types.Struc
source
Sym4state.Utils.get_all_interact_struc_vec — Function

By utilizing the mag_config, this function generates all the Struc objects required for calculating the interaction matrix.

It takes the atomic numbers of magnetic elements stored in mag_num_vec, the index of target atoms denoted by target_idx_vec, and the initial value s_value.

get_all_interact_struc_vec(
    struc::Sym4state.Types.Struc,
    mag_num_vec::Vector{Int64},
    target_idx_vec::Vector{Int64},
    s_value::Float64
) -> Vector{Sym4state.Types.Struc}

Sym4state.Utils.struc_compare — Function

This function examines two Struc objects and determines their approximate equivalence within a tolerance of atol.

It returns a Tuple that includes a boolean value indicating whether they are equal, as well as the corresponding relationships among the atoms in the respective structures.

struc_compare(
    x::Sym4state.Types.Struc,
    y::Sym4state.Types.Struc;
    atol
) -> Tuple{Bool, Dict{Int64, Int64}}

Sym4state.Utils.to_vasp_inputs — Function

With the help of Pymatgen, this function reorganizes the map_vec containing Map objects and the paths of the template INCAR, POSCAR, POTCAR, and KPOINTS files into separate directories representing different magnetic configurations for subsequent DFT calculations.

Additional specific INCAR settings can be provided through the kwargs parameter.

to_vasp_inputs(
    map_vec::Vector{Sym4state.Types.Map};
    incar_path,
    poscar_path,
    potcar_path,
    kpoints_path,
    kwargs...
)
